🔁 Transformers as Limited-Scope RNNs

While transformers lack explicit recurrence, they perform a form of pseudo-recurrence through layered, iterative updates of token embeddings. Each layer “revises” the entire token sequence based on the current state of all tokens — a form of recurrence in depth rather than time. From this angle:

Transformers = wide, shallow recurrence
(full context updated in parallel, but only for a fixed depth)

RNNs = narrow, deep recurrence
(context propagates step by step, potentially indefinitely)

But transformers only simulate recurrence within a limited window, determined by:

The number of layers (as proxy for time steps)

The attention span (number of tokens per query)

The context length cap (due to memory constraints)

Thus, transformers behave like bounded RNNs with shallow, amortized recurrence. They do not:

Maintain internal state across arbitrary input sequences

Form true long-term memory without architectural additions (e.g. recurrence, memory modules)

Learn state transitions over time, unless recurrence is explicitly added (as in Transformer-XL, RetNet, RWKV, etc.)

This connects directly to the critique in “Attention Considered Harmful”:

Transformers are not truly recurrent. They unroll all possible interactions at once, and then discard them. There is no persistent state, no long-range recurrence, no slow feedback. In this sense, transformers are like RNNs that forget every timestep but one. They have no memory — only attention.

🔁 Why This Matters: Relevance vs Recurrence

Real cognition — and RSVP/TARTAN systems — require:

Ongoing relevance tracking

Structured memory and state

Recursive activation patterns, not just flat pairwise comparisons

This is why the Natural Sparsity Principle and Geometric Bayesianism with Sparse Heuristics (GBSH) are superior foundations: they don’t just simulate relevance at one layer or timestep. They evolve it through recursive dynamics and selective memory, like real minds do.
